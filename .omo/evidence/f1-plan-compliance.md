## Blocking Non-Compliance
- FAIL: Plan requires that if a selection crosses non-linked content between linked containers, only text/rect fragments inside registered linked containers are included (`.omo/plans/multi-selection-linked-ranges-plan.md:43`). Implementation clips rect geometry per registered container, but persists `text: selection.toString()` for the linked item (`src/useTextSelection.ts:156` and `src/useTextSelection.ts:159`), which includes non-linked interstitial content. Browser QA confirms the cross-selection item text includes the label text `selectionId = "page-b"` between the two linked `Selection` containers (`.omo/evidence/task-9-manual-qa.md:47`). This is a Must Have gap, so the audit cannot approve.

## Fix Re-Audit

- PASS: Transient linked selected text now uses the filtered linked item text, not the native `selection.toString()`, so the selection popover/active state excludes interstitial non-linked content (`src/useTextSelection.ts:241`).
